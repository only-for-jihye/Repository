# 자기소개

### **On-Premise → AWS Public Cloud 구축 경험 (DevOps)**

2022년 3월부터 SKTelecom TANGO Public Cloud 전환 사업에 참여했습니다. 본격적으로 사업이 시작되기 전, Docker, Kubernetes(K8S), HelmChart에 대해 개인 시간을 할애하여 철저히 준비했습니다. 이 프로젝트의 중점 사항은 Application Modernization이었습니다. Legacy Application을 분석하고 Cloud Native를 위한 설계를 신중히 진행했습니다. 분석 및 설계는 Confluence의 Gliffy Diagram과 표로 세세하게 작성했습니다.

AWS EKS, ECR, MSK, EFS, EBS, ASG, ALB, NLB, S3 등 다양한 AWS 서비스와 리소스를 사용하여 개발했습니다. CodeCommit과 CodeBuild를 CodePipeline으로 구성하고 ArgoCD를 사용하여 CI/CD 파이프라인을 구축했습니다. 이 과정에서 DevOps 환경 구축을 경험했습니다. 특히, 이러한 서비스들을 통합하고 관리하는 과정에서 클라우드 기술의 효율성과 유연성을 깊이 체감할 수 있었습니다.

이 프로젝트를 통해 클라우드 기술의 매력을 더욱 느꼈고, 이를 계기로 2024년 3월에는 Naver Cloud Platform Certified Expert(NCE) 자격증을 취득했습니다. 이 자격증은 저의 클라우드 기술 역량을 공식적으로 증명하는 중요한 성과였습니다. 최근에는 AWS Certified Solutions Architect Associate 자격증에 관심을 갖고 있으며, 이를 통해 클라우드 생태계에 대한 이해를 더욱 심화하고자 합니다.

저의 강점은 Application 분석 및 설계, Public Cloud 구축, 그리고 CI/CD 구성 등 DevOps 전반에 걸친 풍부한 경험입니다. 이 과정에서 새로운 기술을 배우며 업무의 폭을 넓혔습니다. 앞으로도 지속적으로 새로운 기술을 배우며, 전문성을 갖춘 인재로 성장하고 싶습니다. 특히, 변화하는 기술 환경에 발맞추어 지속적인 학습과 성장을 통해 더 큰 가치를 창출하고자 합니다.

### 데이터 파이프라인 운영 경험

SKTelecom TANGO 프로젝트에서 운용-수집 관리 운영 담당자로서 매일 20TB 이상의 데이터를 처리하는 데이터 파이프라인을 운영하고 있습니다. 이 데이터 파이프라인의 역할은 FEP/Gateway를 통해 외부로부터 데이터를 제공받거나 특정 장비/서버에서 데이터를 가져와 이를 가공한 후 각 모듈로 전송하는 것입니다. 데이터 파이프라인 관리에서 중요한 것은 연동 규격을 정의하는 것입니다. 특히 장비가 패치/테스트 등 작업을 진행할 경우, 연동 규격에 따라 데이터의 정상 여부를 판단하여 데이터가 일관성 있고 효율적으로 연동될 수 있도록 지원했습니다. 이 과정에서 벤더사 및 고객사와 협업하며 데이터가 정상적으로 처리될 수 있도록 노력했습니다.

### **커뮤니케이션 스킬 & 책임감을 통한 프로젝트 매니저로서의 성장**
