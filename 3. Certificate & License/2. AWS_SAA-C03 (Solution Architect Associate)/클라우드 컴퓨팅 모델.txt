어플리케이션의 구성

    5. 어플리케이션
    4. OS : Windows/Linux
    3. Computing : CPU + RAM
    2. Storage : HDD/SSD
    1. Network : 랜카드/랜선


- IaaS : Infrastructure as a Service
    - 인프라만 제공
    - OS를 직접 설치하고 필요한 소프트웨어를 개발해서 사용 (4,5만 직접 구성)
    - 즉 가상의 컴퓨터를 하나 임대하는 것과 비슷
    - Ex) AWS EC2

- PaaS : Platform as a Service
    - 인프라 + OS + 기타 프로그램 실행에 필요한 부분 (런타임)
    - 1,2,3,4 + Runtime까지 빌려서 사용, 5만 직접 구성
    - 바로 코드만 올려서 돌릴 수 있도록 구성
    - Ex) Firebase, Google App Engine 등

- SaaS : Software as a Service
    - 1 ~ 5 모두 제공받아 사용
    - 서비스 자체를 제공
    - 다른 세팅 없이 서비스만 이용
    - Ex) Gmail, DropBox, Slack, Google Docs


클라우드 컴퓨팅 배포 모델
    - 공개형(클라우드), public
        - 모든 부분이 클라우드에서 실행
        - 낮은 비용
        - 높은 확장성
    
    - 혼합형(하이브리드), hybrid
        - 폐쇄형과 공개형의 혼합
        - 폐쇄형에서 공개형으로 전환하는 과도기에 사용
        - 혹은 폐쇄형의 백업으로 사용

    - 폐쇄형, private
        - 높은 수준의 커스터마이징 가능
        - 초기 비용이 비쌈
        - 유지 보수 비용이 비쌈
        - 높은 보안
